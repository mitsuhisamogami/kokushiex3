# Sub-Agent Definitions

このディレクトリは、Codex からサブエージェントを起動するときに渡す役割定義を管理します。

## 使い分け

- `planner.md`: 実装前の調査、方針整理、作業分解を担当します。
- `implementer.md`: 方針に沿って、限定された範囲の実装とテスト追加を担当します。
- `code-reader.md`: 既存コードの読解、影響範囲の把握、変更候補の整理を担当します。`coad-reader` と依頼された場合もこの定義を使います。
- `reviewer.md`: レビュー全体の入口です。必要に応じて下記の観点別 reviewer を使います。
- `reviewer/design.md`: 設計、責務分割、Rails らしさ、ドメイン整合性を確認します。
- `reviewer/test.md`: テスト観点、RSpec の網羅性、回帰リスクを確認します。
- `reviewer/security.md`: 認証、認可、入力検証、秘匿情報、CSP、Rack::Attack などを確認します。
- `reviewer/performance.md`: N+1、クエリ効率、メモリ、非同期処理、フロントエンド描画負荷を確認します。

## 共通ルール

- リポジトリの `AGENTS.md` を必ず前提にします。
- 出力は日本語で、結論を先に書きます。
- 指摘はファイルパスと行番号をできるだけ付けます。
- 不明点は推測で断定せず、前提または確認事項として明示します。
- 関係のないリファクタリングや大きな方針変更は提案に留めます。
